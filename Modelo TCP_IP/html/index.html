Información de la Capa 1: Aplicación

Es la capa que proporciona la interfaz entre las aplicaciones que utilizamos para comunicarnos y la red en la cual se transmiten los mensajes, los protocolos se utilizan para intercambiar los datos entre los programas que se encuentran en el origen y el destino. En esta capa se transfiere archivos por medio de la red, se comunican y preparan los datos para la transferencia en texto, imagen o video, estos necesitan diversos servicios para ser procesados, la sintaxis y formato de datos que se transmite, la codificación y conversión de datos para garantizar que los datos del dispositivo de origen se puedan interpretar por medio del dispositivo de destino, la compresión de los datos para que puedan ser descomprimidos en el dispositivo destino, la encriptación de datos para transmitirse y des-encriptarse en el destino. En esta capa existen diversos protocolos para hacer estas funciones, algunos son:
DNS (Domain Name Service): Se utiliza para resolver y traducir los diversos nombres de dominio y sus IPs.
HTTP: Hypertext Transfer Protocol: Se utiliza para transferir archivos que forman paginas web de la world wide web (WWW).
SMTP (Simple MailTransfer Protocol ): Se utiliza para la transferencia de correos electrónicos.
TELNET: Es un protocolo de emulación de terminal, se utiliza para proporcionar acceso remoto a servidores y dispositivos de red como routers.
FTP (File Transfer Protocol): Se utiliza para tranferir archivos entre los sistemas en red.

Información de la Capa 2: Transporte

Su principal tarea es segmentar los datos y brindar el control necesario para re-ensamblar las partes entre distintos nodos de comunicación, nos permite rastrear la comunicación individual entre aplicaciones de origen y destino, segmentación de datos y manejo de cada parte en forma segura, re-ensamble en stream de datos de aplicación para la segmentación de datos por medio del protocolo de control de comunicaciones TCP (Trasmition control protocol). La capa de transporte TCP/IP garantiza que los paquetes lleguen en secuencia y sin errores, al intercambiar la confirmación de la recepción de los datos y retransmitir los paquetes perdidos.

El UDP (user datagram protocol) que también se encuentran en esta capa permite el envió de datagramas a través de la red sin necesidad que se haya establecida antes una conexión debido a que el datagrama incorpora suficiente información de direccionamiento en su cabecera además tampoco confirmación y control de flujo haciendo que los paquetes puedan adelantarse por lo tanto no se sabe si los paquetes se han entregado correctamente.

TCP (Trasmition control protocol): El TCP describe los servicios que segmentan o dividen el flujo de datos.
UDP (User Datagram Protocol): Permite el envió de datagramas a través de la red sin necesidad de una conexión.
IP (Internet Protocol): Es el principal protocolo utilizado en Internet y proporciona el direccionamiento y enrutamiento necesario para de la red.
ICMP (Internet Control Message Protocol): Se utiliza para comunicar mensajes de estado, problemas y errores entre dispositivos en la red.
IGMP (Internet Group Management Protocol): IGMP es un protocolo utilizado en redes IP que admiten la multidifusión (multicast).
ARP (Address Resolution Protocol): ARP es un protocolo de comunicación utilizado para asociar direcciones IP (lógicas) con direcciones MAC (físicas) en una red local.
